강의 시연 진행안과 데모 시나리오

Showcase.md

문서 목적

이 문서는 강의/세미나에서 HWPX Pipeline 플러그인을 시연하기 위한 상세 진행안입니다.

  • 대상: 교사, 공공기관 실무자, 문서 자동화 관심 개발자
  • 권장 시연 시간: 25~40분
  • 핵심 메시지:
  1. HWPX와 Markdown을 오가며 문서 생산성을 높일 수 있다.
  2. AI를 안전하게 붙여도 문서 통제권을 유지할 수 있다.
  3. 실무 운영에 필요한 충돌 처리, 템플릿 관리, 문제 대응까지 준비되어 있다.

1. 시연 전 준비

1.1 환경 체크리스트

  • Obsidian 최신 버전 (Desktop)
  • 플러그인 설치 완료 (Install.md 참고)
  • 샘플 파일 준비:
  1. samples/교실환경개선_추진계획.hwpx
  2. 추가 시연용 HWPX 파일 2~3개 (일괄 임포트용)
  • 네트워크 확인 (AI 시연 시 필요)
  • AI API 키 설정 (세션 저장 권장)

1.2 시연용 권장 설정

설정 → HWPX Pipeline

  • 파일 충돌 처리: 매번 물어보기
  • 임포트 미리보기: ON
  • 익스포트 미리보기: ON
  • 익스포트 결과 전달: Vault 저장 + 다운로드
  • 기본 컨텍스트: 선택 텍스트
  • frontmatter 제외: ON
  • 최대 컨텍스트 길이: 12000

1.3 발표자 멘트 준비

  • 도입 멘트 예시:
    • "오늘은 HWPX 원본 양식을 버리지 않고 Markdown 기반 협업/자동화를 붙이는 방법을 보여드리겠습니다."
  • 전환 멘트 예시:
    • "이제 문서 변환이 아니라 운영 단계에서 필요한 안전장치를 보겠습니다."

2. 시연 시나리오 (권장 30분)

2.1 0~3분: 문제 정의

  • 현재 실무 문제 제시:
  1. HWPX 원본 양식 유지가 어렵다.
  2. AI 사용 시 민감정보 노출이 불안하다.
  3. 파일 충돌/버전 관리가 번거롭다.
  • 해결 방향 제시: 임포트 → 편집/AI → 익스포트 파이프라인

2.2 3~10분: 단일 HWPX 임포트

  • 실행:
  1. 커맨드 팔레트에서 HWPX 파일 임포트 (→ Markdown) 실행
  2. 교실환경개선_추진계획.hwpx 선택
  3. 임포트 미리보기 확인 후 저장
  • 강조 포인트:
  1. 제목/본문/표가 Markdown으로 변환됨
  2. frontmatter에 출처 정보(hwpx_pipeline)가 남아 추적 가능
  3. 템플릿 폴더(_hwpx_templates)에 원본 저장됨

2.3 10~17분: AI 문서 작성 도우미

  • 실행:
  1. 기존 문서에서 일부 문단 선택
  2. AI 문서 작성 도우미 실행
  3. 참고 컨텍스트: 선택 텍스트 확인
  4. 프롬프트 입력 후 생성
  • 예시 프롬프트:
    • "위 내용을 기준으로 예산 집행 근거를 공문서 톤으로 5문장 추가해줘"
  • 강조 포인트:
  1. 선택 텍스트만 전송 가능
  2. frontmatter 제외/길이 제한 정책으로 데이터 최소 전송
  3. 생성 결과를 현재 커서에 삽입

2.4 17~23분: 템플릿 관리 + 연결

  • 실행:
  1. HWPX 템플릿 관리 커맨드 실행
  2. 템플릿 목록 확인
  3. 현재 노트를 특정 템플릿에 연결 (현재 노트 연결)
  • 강조 포인트:
  1. 템플릿 교체/삭제/재연결 가능
  2. 노트와 원본 양식의 연결이 문서 내에서 관리됨

2.5 23~28분: HWPX 익스포트

  • 실행:
  1. 현재 노트를 HWPX로 내보내기 실행
  2. 익스포트 미리보기 확인
  3. Vault 저장 + 다운로드 결과 확인
  • 강조 포인트:
  1. 템플릿 기반 재출력으로 형식 보존
  2. 동일 파일명 충돌 시 정책대로 처리됨

2.6 28~30분: 일괄 임포트

  • 실행:
  1. 여러 HWPX 파일 일괄 임포트 실행
  2. 2~3개 파일 선택
  3. 결과 Notice 확인 (성공/건너뜀/실패 건수)
  • 강조 포인트:
  1. 실무 초기 전환 시 대량 처리 유리
  2. 충돌 정책과 함께 운영 안정성 확보

3. 시연 중 질문 유도 포인트

  • "민감한 문서는 AI에 어디까지 보내야 안전할까요?"
  • "기존 HWPX 양식을 바꿔야 하나요, 그대로 써도 되나요?"
  • "여러 명이 동시에 편집할 때 충돌은 어떻게 처리하나요?"

4. 장애 대비 플랜

4.1 인터넷 불가 (AI 시연 실패)

  • 대체 진행:
  1. AI 섹션은 설정/보안 옵션 설명 위주로 전환
  2. 임포트/익스포트/템플릿 관리 중심으로 시연 지속

4.2 파일 충돌 반복 발생

  • 대체 진행:
  1. 파일 충돌 처리를 새 버전 저장으로 변경
  2. 충돌 관리 UX 시연 포인트로 전환

4.3 잘못된 템플릿 연결

  • 대체 진행:
  1. 템플릿 관리 모달에서 즉시 재연결
  2. source_file 업데이트 동작 설명

5. 강의 마무리 체크

  • 데모 종료 후 강조할 실무 적용 순서:
  1. 설치 (Install.md)
  2. 사용자 교육 (UserGuide.md)
  3. 운영 정책 확정 (충돌/AI 전송/템플릿 폴더)
  4. 내부 배포/개발 확장 (Project.md, Developer.md)

  1. UserGuide.md
