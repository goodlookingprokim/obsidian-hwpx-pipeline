강의 시연 진행안과 데모 시나리오

Showcase.md

문서 목적

이 문서는 강의 현장에서 쉽게 진행되지만 임팩트는 크게 보이도록 설계한 시연 스크립트입니다.

핵심 원칙:

  1. PART A. 하이라이트 홍보로 관심을 먼저 잡습니다.
  2. PART B. 시연 파트로 시작부터 종료까지 같은 품질로 진행합니다.
  3. 어떤 강사가 진행해도 같은 흐름으로 마무리되도록 문장을 고정 제공합니다.

PART A. 하이라이트 홍보 파트

A1. 한 줄 포지셔닝

HWPX Pipeline은 기존 HWPX 양식을 버리지 않고, Markdown + AI 생산성을 즉시 붙이는 실무형 플러그인입니다.

양식 보존

HWPX를 가져오고 다시 내보낼 때 템플릿 기반으로 형식을 유지합니다.

Import + Template Export

작업 속도

임포트 → AI 보강 → 익스포트를 하나의 흐름으로 시연할 수 있습니다.

Fast Demo Loop

운영 안정성

충돌 정책, 미리보기, 경로 검증으로 실무 리스크를 제어합니다.

Safe by Default

쉽다 빠르다 실무형

A2. 90초 하이라이트 홍보 스크립트 (그대로 사용)

  1. "우리는 기존 문서 양식을 버리지 않습니다. 바꾸는 것은 작성 속도입니다."
  2. "이 플러그인은 HWPX를 Markdown으로 가져오고, 편집 후 다시 HWPX로 내보냅니다."
  3. "AI를 붙여도 통제가 가능합니다. 선택 텍스트만 보내고 길이도 제한합니다."
  4. "충돌 정책과 미리보기로 실수 비용을 줄입니다. 현업에서 바로 쓸 수 있습니다."
  5. "오늘은 복잡한 기능 설명이 아니라, 실제 보고서 작성 흐름으로 보여드리겠습니다."

A3. 홍보 멘트 하이라이트 카드

강조 주제 강력 멘트
기존 양식 보호 "양식은 그대로, 작성 방식만 현대화합니다."
시간 절감 "문서 변환 + 보강 + 배포를 한 흐름으로 끝냅니다."
보안/통제 "AI는 쓰되, 보내는 데이터는 우리가 통제합니다."
운영성 "시연용 기능이 아니라 배포 가능한 운영 기능입니다."

A4. 하이라이트 파트 운영 팁

  1. 첫 30초 안에 양식 보존을 반드시 말합니다.
  2. 그 다음 30초는 속도, 마지막 30초는 통제/운영 순서로 말합니다.
  3. 용어보다 결과를 말합니다: "무엇이 쉬워졌는가", "무엇이 안전해졌는가".

PART B. 시연 파트 (강의 개요 + 강사 스크립트)

B1. 강의 개요 (시작부터 종료까지 고정 흐름)

구간 시간 목표 강사 행동 완료 기준
오프닝 0~2분 문제-해결 프레임 고정 홍보 스크립트 5문장 낭독 관객이 "양식 유지 + 자동화"를 이해
임포트 2~6분 원본 문서 전환 체감 단일 HWPX 임포트 실행 Markdown 노트 생성 확인
AI 보강 6~10분 생산성 + 통제 동시 강조 선택 텍스트 기반 생성 생성 결과 삽입 + 전송 범위 설명
템플릿 관리 10~13분 운영 신뢰 확보 템플릿 연결/교체 동작 시연 source_file 연결 흐름 확인
익스포트 13~17분 실무 포맷 복귀 확인 HWPX 익스포트 실행 다운로드 또는 Vault 결과 확인
마무리 17~20분 적용 행동 유도 클로징 3문장 + 안내 링크 설치/운영/개발 다음 단계 제시

B2. 강사 사전 준비 체크리스트

  1. Obsidian Desktop 실행 및 플러그인 활성화 확인
  2. 샘플 파일 준비:
    • samples/교실환경개선_추진계획.hwpx
  3. AI 데모용 API 키 준비 (세션 저장 권장)
  4. 설정 확인:
    • 파일 충돌 처리: 매번 물어보기
    • 임포트/익스포트 미리보기: ON
    • 기본 컨텍스트: 선택 텍스트
    • frontmatter 제외: ON
    • 최대 컨텍스트 길이: 12000
  5. 시연 전 단축키 확인:
    • Mod + Shift + H: HWPX 3단계 워크플로우 코치 열기
    • Mod + Shift + L: HWPX 외부 LLM 프롬프트 키트 열기
    • Mod + Shift + T: HWPX 템플릿 관리

B3. 리허설/시연 명령어

B3-1. 시연 전 1분 체크 (개별 실행)

git clone https://github.com/goodlookingprokim/obsidian-hwpx-pipeline.git
cd obsidian-hwpx-pipeline
npm install
npm run build

B3-2. 리허설 품질 체크 (동시 실행 가능)

npm run build
npm test

B4. 강사 스크립트 원문 (복붙 진행용)

0단계. 시작 멘트 (0:00~1:00)

강사 멘트:

  1. "오늘은 HWPX 문서 체계를 유지하면서 작성 속도를 높이는 흐름을 보여드립니다."
  2. "핵심은 간단합니다. 가져오고, 보강하고, 다시 배포합니다."

강사 행동:

  1. 문서 포털 Showcase 페이지를 열어 오늘 진행 순서를 보여줍니다.
  2. "20분 안에 끝나는 실무 시연"이라고 시간 범위를 먼저 고지합니다.

1단계. 문제 정의 + 가치 제안 (1:00~2:00)

강사 멘트:

  1. "기존 문제는 세 가지입니다. 양식 유지, 작성 속도, 운영 안정성."
  2. "HWPX Pipeline은 이 세 가지를 동시에 해결하도록 설계되었습니다."

강사 행동:

  1. 하이라이트 카드(양식 보존/작업 속도/운영 안정성)를 위에서 아래로 읽습니다.

2단계. 단일 임포트 시연 (2:00~6:00)

강사 멘트:

  1. "이제 원본 HWPX를 Markdown으로 가져오겠습니다."
  2. "결과 노트에서 메타데이터와 본문 구조를 함께 보겠습니다."

강사 행동:

  1. 커맨드 팔레트 열기
  2. HWPX 파일 임포트 (→ Markdown) 실행
  3. 교실환경개선_추진계획.hwpx 선택
  4. 미리보기 확인 후 저장
  5. 생성 노트에서 hwpx_pipeline 정보 확인

완료 확인 멘트:

  1. "원본은 템플릿으로 보관되고, 편집은 Markdown에서 빠르게 진행됩니다."

3단계. AI 보강 시연 (6:00~10:00)

강사 멘트:

  1. "AI는 문서 전체가 아니라 필요한 부분만 보냅니다."
  2. "그래서 빠르고, 불필요한 전송을 줄일 수 있습니다."

강사 행동:

  1. HWPX 외부 LLM 프롬프트 키트 열기 실행
  2. 문단 일부 선택
  3. 프롬프트 템플릿 복사 후 외부 LLM에 입력
  4. 프롬프트 입력 예시:
    • "위 내용을 기준으로 실행 계획 핵심 요약 4문장 작성"
  5. 결과를 Markdown에 반영
  6. (선택) AI 문서 작성 도우미로 동일 흐름 재현

완료 확인 멘트:

  1. "AI를 쓰되, 통제권은 사용자가 갖습니다."

4단계. 템플릿 관리 시연 (10:00~13:00)

강사 멘트:

  1. "문서와 양식 연결이 명확해야 팀 운영이 안정적입니다."

강사 행동:

  1. HWPX 템플릿 관리 실행
  2. 템플릿 목록 확인
  3. 현재 노트 연결 동작 시연

완료 확인 멘트:

  1. "템플릿 연결 상태를 사람이 바로 이해하고 관리할 수 있습니다."

5단계. HWPX 익스포트 시연 (13:00~17:00)

강사 멘트:

  1. "이제 결과를 다시 HWPX로 내보내겠습니다."
  2. "현업 제출 포맷으로 복귀하는 순간이 핵심입니다."

강사 행동:

  1. 현재 노트를 HWPX로 내보내기 실행
  2. 미리보기 확인
  3. Vault 저장 + 다운로드 선택
  4. 산출물 확인

완료 확인 멘트:

  1. "양식은 유지되고, 내용은 더 빠르게 고도화됩니다."

6단계. 클로징 (17:00~20:00)

강사 멘트:

  1. "오늘 본 흐름은 단순합니다. 가져오기, 보강하기, 배포하기."
  2. "핵심은 양식 보존 + 생산성 + 운영 안정성을 동시에 확보하는 것입니다."
  3. "바로 적용하려면 설치 문서부터 시작하시면 됩니다."

강사 행동:

  1. Install.md, UserGuide.md, TroubleShooting.md 링크를 순서대로 안내
  2. 질문을 받되, 질문이 길어지면 다음 답변 템플릿 사용

질문 응답 템플릿:

  1. "좋은 질문입니다. 짧게 결론부터 말씀드리면..."
  2. "실행 방법은 문서의 해당 섹션에서 바로 재현 가능합니다."

B5. 시연 중 강조할 강점 체크포인트

  1. 양식 보존: "원본 HWPX와 템플릿 연결 유지"
  2. 속도: "임포트-보강-익스포트 한 흐름"
  3. 안전성: "선택 텍스트 전송 + 길이 제한 + 충돌 정책"

B6. 장애가 생겨도 완주하는 대체 시나리오

  1. AI 네트워크 오류:
    • "AI 구간은 건너뛰고 임포트/익스포트 흐름으로 계속 진행하겠습니다."
  2. 파일 충돌 반복:
    • 파일 충돌 처리를 새 버전 저장으로 변경 후 재시도
  3. 템플릿 오연결:
    • HWPX 템플릿 관리에서 현재 노트 연결로 복구
