문서 체계 운영 계획과 점검 항목

DocumentationPlan.md

1. 목적

플러그인 도입/시연/개발/운영에 필요한 문서를 체계적으로 관리하기 위한 계획 문서입니다.

2. 문서 작성 범위

2.1 필수 문서

  • Showcase.md: 강의/데모 시연 시나리오
  • Project.md: 유사 프로젝트 기획/개발 프레임워크
  • Developer.md: 개발자 유지보수 매뉴얼
  • UserGuide.md: 사용자 작업 매뉴얼
  • TroubleShooting.md: 오류 대응 노하우
  • Install.md: 설치/업데이트/제거 가이드

2.2 보강 문서

  • README.md: 문서 허브 및 빠른 진입점
  • CHANGELOG.md: 릴리스 변경 이력

3. 작성 원칙

  1. 사용자 관점 문서와 개발자 관점 문서를 분리한다.
  2. 모든 문서는 실제 커맨드/설정명 기준으로 작성한다.
  3. 문제 해결 문서는 증상-원인-해결 순서를 유지한다.
  4. 릴리스마다 문서 변경 여부를 점검한다.

4. 진행 상태

  • 문서 범위 확정
  • Showcase.md 작성
  • UserGuide.md 작성
  • Install.md 작성
  • Project.md 작성
  • Developer.md 작성
  • TroubleShooting.md 작성
  • 정기 문서 점검 자동화 (추가 과제)

5. 정기 점검 체크리스트

릴리스 전

  1. 신규 기능이 사용자 문서에 반영되었는가
  2. 설정 변경이 개발자 문서에 반영되었는가
  3. 설치/업데이트 절차가 최신인가
  4. 알려진 이슈가 TroubleShooting에 반영되었는가

릴리스 후

  1. 시연 문서가 실제 동작과 일치하는가
  2. 사용자 FAQ가 업데이트되었는가
  3. 개발자 온보딩 절차가 재현 가능한가

6. 다음 개선 과제

  1. 문서 버전 태깅(v1.1, v1.2)
  2. 스크린샷/캡처 이미지 추가
  3. 사내 배포용 PDF 패키징
